Lecture 17: Git & GitHub Basics with Bash

Summary

المحاضرة دي كانت مقدمة قوية عن نظام التحكم في الإصدارات Git وعن موقع GitHub اللي بنستخدمه لاستضافة المشاريع ومشاركتها أونلاين. اتعلمنا إزاي نستخدم Git Bash كـ terminal علشان نكتب أوامر تساعدنا نتحكم في المشروع، نحفظ التغييرات، ونرفعها على GitHub. الهدف من المحاضرة إنك تبدأ تستخدم Git في شغلك وتحافظ على كل إصدارات المشروع بسهولة.

Key Points

  • ما هو Git؟
    Git هو نظام لإدارة الإصدارات (Version Control System). فكرته الأساسية إنه بيحتفظ بتاريخ كل تعديل بتعمله في الكود، ويسمحلك ترجع لأي نسخة سابقة في أي وقت.
    • بيساعدك تشتغل بأمان من غير ما تخاف تضيع كودك.
    • مفيد جدًا لما تشتغل في فريق.
  • ما هو GitHub؟
    GitHub هو موقع بيخزن المشاريع اللي بتستخدم Git على الإنترنت، وبيخليك تشاركها أو تشتغل عليها مع ناس تانية. ممكن تعتبره زي “سوشيال ميديا للمبرمجين” 😄
    • كل مشروع بيبقى في "Repository".
    • ممكن ترفع مشاريعك الخاصة (Private) أو العامة (Public).
  • البدء باستخدام Git Bash:
    Git Bash هو تطبيق بيوفر لك بيئة أوامر شبيهة بالـ Linux داخل Windows. بنستخدمه علشان نكتب أوامر Git ونتعامل مع الملفات. أهم الأوامر الأساسية:
    • pwd → بيعرض المسار الحالي (المكان اللي انت فيه).
    • ls → بيعرض الملفات والمجلدات.
    • cd foldername → يفتح مجلد معين.
    • cd .. → يرجع خطوة للخلف.
    • clear → ينظف الشاشة.
  • إنشاء مشروع جديد وربطه بـ Git
    لما تفتح مجلد المشروع في Git Bash، اكتب الأوامر دي بالترتيب:
    1. git init → يهيئ المجلد ويخليه Git repository.
    2. git status → يوريك الملفات المعدلة أو الجديدة.
    3. git add . → يضيف كل الملفات للـ staging area.
    4. git commit -m "Initial commit" → يحفظ التغييرات برسالة.
    كده انت خزّنت أول نسخة من مشروعك داخل Git.
  • الربط مع GitHub
    بعد ما تعمل Repository على GitHub (خالي من الملفات)، ارجع لـ Git Bash واكتب:
    1. git remote add origin https://github.com/username/repo.git
    2. git branch -M main → لتحديد الفرع الرئيسي.
    3. git push -u origin main → يرفع الملفات إلى GitHub.
    كده المشروع بقى أونلاين على حسابك.
  • أوامر مهمة جدًا لازم تعرفها:
    • git log → يعرض سجل كل الـ commits.
    • git diff → يوريك التغييرات اللي حصلت في الملفات.
    • git pull → يسحب آخر تحديث من GitHub.
    • git clone [repo link] → ينزل مشروع من GitHub لجهازك.
  • التعامل مع الأخطاء:
    لو Git Bash طلب منك تسجيل اسمك وإيميلك لأول مرة، لازم تضيفهم كده:
    git config --global user.name "Your Name"
    git config --global user.email "your@email.com"

Notes

💡 استخدم git status دايمًا قبل أي push علشان تعرف حال الملفات.
⚙️ كل commit لازم يكون له رسالة واضحة تعبر عن التغييرات اللي حصلت.
📂 متنساش تعمل .gitignore علشان تمنع رفع ملفات النظام أو المكتبات الضخمة.
🧠 Git مش بس لحفظ الكود — ده أداة بتخليك تتحكم في تاريخ مشروعك بالكامل.
